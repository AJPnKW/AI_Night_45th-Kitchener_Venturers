## Topic Expansion

### 1. Intro to AI: What AI Is and Is Not

- Purpose: remove hype and give the group a stable mental model.
- Audience fit: mixed-ability Venturer learners with uneven prior exposure.
- Delivery mode: 5-minute live explanation plus one concrete example.
- Time required: 5 minutes in the 90-minute session, 3 minutes in the 1-hour version.
- Facilitator preparation: choose one helpful use case and one obvious limit case.
- Learner steps: compare a normal tool with an AI helper and answer one short reflection.
- Exercise sequence: show -> compare -> question.
- Work effort: low.
- Expected outputs: learners understand AI as a helper, not an authority.
- Risks/misuse: hype framing, anthropomorphism, or vague “AI magic” talk.
- Validation notes: supported by Code.org intro content and OpenAI help content.
- Adaptation notes: keep examples practical and youth-appropriate.

### 2. Intro to ChatGPT: How to Use It Well

- Purpose: give each learner one repeatable success pattern.
- Audience fit: beginners ages 15 to 18.
- Delivery mode: live demo followed by a same-task common first prompt.
- Time required: 10 minutes combined.
- Facilitator preparation: prepare one weak prompt and one improved version.
- Learner steps: run the starter prompt, compare output, improve once.
- Exercise sequence: ask -> check -> improve -> re-run.
- Work effort: low to medium.
- Expected outputs: one working prompt-result pair per learner.
- Risks/misuse: copy-paste behavior without checking results.
- Validation notes: aligned to prompt guidance and language-model interaction references.
- Adaptation notes: model exact steps and avoid jargon.

### 3. Teen-Safe Use / Parent Controls / Approvals

- Purpose: make safety and supervision visible before independent work begins.
- Audience fit: participants, parents, leaders, facilitator.
- Delivery mode: pre-work page, parent summary, and short spoken reminder.
- Time required: 5 minutes in session plus pre-session review.
- Facilitator preparation: know the approval path and the minimum-data rule.
- Learner steps: avoid personal data, use first-name-only patterns where needed, ask when unsure.
- Exercise sequence: compare okay vs not-okay prompt content.
- Work effort: low.
- Expected outputs: fewer risky prompts and clearer parent confidence.
- Risks/misuse: drifting into vague warnings or over-legalized wording.
- Validation notes: anchored to the Teen AI Literacy Blueprint and OpenAI Academy K-12 guidance.
- Adaptation notes: keep tone calm, specific, and practical.

### 4. Prompting, Rulesets, Personalization, and Settings

- Purpose: show that better instructions produce better outputs.
- Audience fit: beginners ready for one layer deeper after early success.
- Delivery mode: mini-lesson plus optional extension card.
- Time required: 10 to 15 minutes.
- Facilitator preparation: prepare one example using audience, tone, and output format.
- Learner steps: add context, set a format, compare default vs guided output.
- Exercise sequence: baseline -> improve -> compare.
- Work effort: medium.
- Expected outputs: one reusable prompt template.
- Risks/misuse: too much settings depth too early.
- Validation notes: supported by memory and custom-instructions references.
- Adaptation notes: make settings depth optional.

### 5. Hands-On Beginner Activities for Ages 15-18

- Purpose: make the session practical and personal.
- Audience fit: mixed-skill older youth.
- Delivery mode: project menu with 3 to 4 guided choices.
- Time required: 35 to 45 minutes in the 90-minute version.
- Facilitator preparation: curate limited pathways with clear labels.
- Learner steps: choose a pathway, build, refine, prepare one thing to share.
- Exercise sequence: choose -> prompt -> test -> refine -> share.
- Work effort: medium.
- Expected outputs: one mini-project and one share-back artifact.
- Risks/misuse: too many choices causing paralysis.
- Validation notes: supported by tool walkthroughs and writing-with-AI teaching examples.
- Adaptation notes: label pathways by difficulty and keep choices limited.

### 6. Scout / Youth-Group Adaptation

- Purpose: make the workshop relevant to Venturer contexts without becoming childish.
- Audience fit: older youth in a volunteer-led scouting environment.
- Delivery mode: scouting-relevant examples integrated into the normal pathways.
- Time required: integrated throughout the session.
- Facilitator preparation: prepare examples tied to planning, service, communication, and activity ideas.
- Learner steps: optionally choose a scout-relevant project lane.
- Exercise sequence: use the standard prompt flow with scout-themed output.
- Work effort: low.
- Expected outputs: easier prompt ideas and higher engagement.
- Risks/misuse: confusion between unofficial support content and official organizational messaging.
- Validation notes: based on workshop goals and parent-confidence needs.
- Adaptation notes: keep disclaimers visible where appropriate.

### 7. Intro to Codex / Beginner Coding-Agent Awareness

- Purpose: explain AI coding help without implying autonomy or trustworthiness.
- Audience fit: learners who ask about coding or already have some coding exposure.
- Delivery mode: optional short extension segment.
- Time required: 5 to 8 minutes.
- Facilitator preparation: choose one tiny example such as a simple HTML tweak or checklist script.
- Learner steps: ask for help, inspect the result, test it, fix it if needed.
- Exercise sequence: ask -> inspect -> test -> verify.
- Work effort: medium for facilitator, low for learners.
- Expected outputs: a grounded understanding that AI can draft but still needs human review.
- Risks/misuse: overtrust, security blind spots, magic-tool framing.
- Validation notes: supported by local OpenAI agent and Codex references.
- Adaptation notes: keep this optional unless the group is moving quickly.

### 8. What Beginners Usually Are Not Told

- Purpose: inoculate learners against overtrust and confusion.
- Audience fit: all participants.
- Delivery mode: short reminders woven through the session rather than one heavy warning block.
- Time required: about 5 minutes total across the session.
- Facilitator preparation: prepare one example for each issue.
- Learner steps: compare, question, verify, revise.
- Exercise sequence: spot a problem in an output and improve the prompt or check path.
- Work effort: low.
- Expected outputs: healthier caution and better habits.
- Risks/misuse: making the tool feel pointless instead of usable-with-care.
- Validation notes: anchored to bias, truthfulness, and generated-links references.
- Adaptation notes: pair each warning with one usable habit.

## Activity Bank

### Activity 1: First Prompt Win

- Purpose: give everyone an early successful result.
- Audience fit: complete beginners.
- Delivery mode: whole-group guided task.
- Time required: 5 to 7 minutes.
- Facilitator preparation: choose one common starter prompt and one improved version.
- Learner steps:
  1. Open ChatGPT.
  2. Run the starter prompt.
  3. Read the result.
  4. Improve the prompt once.
  5. Compare the difference.
- Expected outputs: a first successful result for each learner.
- Adaptation notes: provide a printed fallback prompt if device friction appears.

### Activity 2: Project Pathway Menu

Pathway A: Event planner helper  
Pathway B: Scout activity idea builder  
Pathway C: Short script or message draft  
Pathway D: Learn-something-quick research helper

Each pathway should keep the same shape:

1. choose a prompt goal
2. run a first version
3. improve the result
4. prepare one thing to share

### Activity 3: Stuck Recovery Ladder

- Purpose: keep learners moving when they freeze or get weak outputs.
- Delivery mode: facilitator card or visible on-screen support ladder.
- Time required: 2 to 3 minutes.
- Learner steps:
  1. say what you want in one sentence
  2. say who it is for
  3. say what format you want
  4. ask for three options instead of one
  5. ask for a reset prompt from a leader or facilitator

### Activity 4: Fast Learner Extension

- Purpose: keep quick learners engaged without splitting the room.
- Delivery mode: optional extension card.
- Time required: 5 to 15 extra minutes.
- Example extension moves:
  - add tone control
  - add format control
  - compare two prompt styles
  - convert output into a checklist, table, or script

### Activity 5: Parent Co-Review Task

- Purpose: connect home awareness to workshop readiness.
- Delivery mode: pre-work.
- Time required: 5 minutes.
- Learner steps:
  1. review the participant pre-work page with a parent or guardian
  2. confirm account and device readiness
  3. review the no-personal-data rule

## Teen-Safe Use and Parent Controls

### What the Youth Should Hear

- AI can help, but it can be wrong.
- Do not put private details into prompts.
- If you are unsure whether something is okay to enter, ask first.
- The workshop is supervised and learning-focused.

### What Parents Should Hear

- The activity is structured and supervised.
- The youth are learning how to use the tool responsibly, not just experimenting without boundaries.
- Setup help may be needed at home before the event.
- Only minimal readiness information should be collected.

### What Leaders Should Watch For

- learners typing personal details
- learners getting stuck and frustrated
- learners moving too fast and drifting off-task
- learners needing a predictable next step

### What the Facilitator Should Control

- shared prompt examples
- explanation of safe use
- room tempo and transitions
- escalation path when a learner is unsure

## Beginner-Safe Codex Framing

- Codex is a helper, not an autopilot.
- It can produce useful drafts quickly.
- It can also produce errors or mismatched code.
- Humans still inspect, test, and decide.

Good beginner examples:

- a tiny event-checklist page
- a short HTML layout tweak
- a simple planning-helper script

## What Beginners Are Not Told

### Key missing truths

1. AI can sound sure and still be wrong.
2. The first answer is usually a draft, not a final result.
3. Bias and gaps still show up.
4. Privacy is still the user’s responsibility.
5. Convenience can lead to overtrust.

### Habit pairings

- wrong-but-confident -> verify one key claim
- weak prompt -> add goal, audience, and format
- privacy risk -> remove names and specifics
- biased answer -> ask for another perspective
- overtrust -> compare with another source
