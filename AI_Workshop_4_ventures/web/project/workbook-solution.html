Participant Workbook Solution

Recommended individualized participant workflow using Pages, Google Forms, Sheets, and optional Apps Script.

Why the current workbook is not enough on its own

The current local browser workbook is useful for live note-taking, but it does not fully behave like an LMS. If each participant needs an individual workflow with saved responses, prompt selection, adaptive follow-up, and completion capture, the solution needs a structured forms layer rather than GitHub Pages alone.

Recommended tool pattern

NeedRecommended toolWhy it fits
Public instructions, exercises, examples, and referencesGitHub PagesLow-friction public front end with good browsing and printing support.
Per-participant guided workflowGoogle Forms with section branchingProvides structured interaction, required fields, section logic, and response capture.
Response storeGoogle SheetsEasy operational visibility and exports.
Participant-specific linksGoogle Forms prefilled URLsLets each learner start from a personalized or pathway-specific form without building a custom app.
Optional end-of-flow email or PDF automationGoogle Apps ScriptAdds the missing LMS-like completion layer without forcing a full backend into the public site.

Suggested workflow

StepExperience
1. Entry pointParticipant starts from the public site and opens the organizer-issued workbook form link.
2. IdentificationThe form captures participant name or alias, role, group, and pathway assignment.
3. Guided sectionsThe form walks the learner through prompt improvement, image prompt, reflection, and follow-up.
4. Prompt selectionThe learner chooses from starter prompt options or a guided level path.
5. Note captureThe learner records what they changed, what result they got, and what gap remained.
6. Adaptive progressionBranching sends learners to Guided, Build, Remix, or Stretch follow-up items.
7. Completion outputThe response lands in Sheets and can optionally trigger an Apps Script summary email or PDF workflow.

Right-sized recommendation

The practical next step is not to build a full LMS. It is to keep GitHub Pages as the public front end, add a participant workbook Google Form with section branching and prefilled links, and only add Apps Script where real workflow automation is needed.
