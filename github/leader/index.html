Leader Guide

Browser-first support guide for leaders.

Leader support role

Keep the night calm, browser-first, and low-choice. Help youth get to the right page, keep them powered, and route issues into the fallback plan instead of derailing the session with advanced setup.

What leaders help with

  • Chargers and power access
  • Getting youth to the correct page
  • Noticing who is stuck
  • Using fallback routes early

Access model

Participant and parent pages are public. Leader and facilitator materials are separate by circulation and role, not by true security controls inside GitHub Pages.

Session-night logistics

SituationLeader action
Late arrivalOpen the youth page, confirm charger access, and route to the current exercise.
Device troubleUse paired work, presenter-led participation, or the fallback prompt cards before touching advanced setup.
Internet issueShift to group demo visibility, printed prompt cards, and verbal reflection.
Learner overloadReduce choices, suggest Guided, and narrow to one prompt only.

What not to derail into

  • Do not push VM setup as the main route.
  • Do not turn the night into coding troubleshooting.
  • Do not create extra tool choices if the browser-first path is working.
